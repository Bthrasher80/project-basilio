1. Purpose & Security Objective

The purpose of this control is to ensure secure, auditable, and operationally realistic credential management across all Project Basilio systems.

This practice is designed to mirror real-world SOC and infrastructure environments by enforcing:

Centralized credential storage

Unique credentials per system

Clear separation between infrastructure access and identity accounts

MFA-protected access to sensitive secrets

This reduces credential sprawl, prevents reuse, and supports safe operational workflows during administration, investigations, and incident response.

2. Credential Management Platform
Password Manager

Platform Used: Bitwarden
Deployment Model: Cloud-hosted (Free Tier)
Protection: Multi-Factor Authentication (MFA) enabled

Bitwarden was selected due to:

Broad enterprise adoption

Strong security model

MFA support

Clean separation of credentials via collections

Low operational overhead for a lab environment

Note: While the free tier limits the number of collections, this constraint was intentionally handled through naming conventions and access modeling rather than additional tooling.

5. Service Account Discipline

Dedicated service accounts are used instead of reusing administrator credentials.

Key principles:

One service account per integration

Least privilege where possible

Clear purpose defined per account

Stored and tracked centrally

This directly supports:

Authenticated vulnerability scanning

Secure log ingestion

Future access reviews and remediation workflows

6. Credential Hygiene Standards

The following hygiene standards are enforced:

No password reuse across systems

All credentials generated by the password manager

Long, random passwords used consistently

MFA required for password manager access

No credentials stored in plaintext files, scripts, or notes

These practices reduce the likelihood of:

Credential compromise

Lateral movement

Accidental exposure during documentation or troubleshooting
